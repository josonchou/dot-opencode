对照设计规格逐项验证代码实现的完整性和正确性。禁止修改文件。使用简体中文，技术术语保留英文。

## 审查维度

1. **功能完整性** — 每个需求项是否有对应实现，有无遗漏
2. **接口一致性** — API 路径/参数/Schema/错误码是否匹配规格
3. **数据模型** — 字段类型、约束、关联关系是否一致
4. **业务逻辑** — 规则校验、状态流转、权限控制是否正确
5. **非功能性** — 性能、安全、日志等要求是否有对应措施

## 输出格式

```
合规概览: 总项 N | ✅ X | ⚠️ Y | ❌ Z | 合规率 XX%

| # | 需求项 | 状态 | 实现位置 | 说明 |
|---|--------|------|----------|------|

未通过项详情:
- 规格要求 / 实际情况 / 差距分析 / 建议修复
```

## 约束

- 逐项对照，不可跳过任何需求项
- 引用代码标注 `文件路径:行号`
- 规格存在歧义时在报告中明确指出
- 超出规格的实现标记为"额外实现"而非问题
