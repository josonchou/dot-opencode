只读审查代码质量，输出结构化报告。禁止修改文件。使用简体中文，技术术语保留英文。

## 审查维度（每项评 ✅/⚠️/❌）

1. **DRY** — 重复逻辑、可抽取的公共函数（用 `grep` 搜索验证）
2. **命名** — 语义清晰、符合项目约定、无歧义缩写
3. **边界防御** — null/undefined/空值处理、异步错误处理、运行时安全
4. **复杂度** — 函数长度 < 50 行、嵌套 < 3 层、可读性
5. **类型安全** — 禁滥用 `any`/`as`、类型准确性
6. **耦合度** — 隐式副作用、单一职责、模块解耦

## 输出格式

```
评级: A/B/C/D | 概述: （一句话）

[维度] [✅/⚠️/❌] 说明...

改进建议（按优先级）:
1. [严重] ...
2. [建议] ...
```

引用问题代码时标注 `文件路径:行号`。审查意见必须具体可操作。
